SPRING BOOT ARCHITECTURE - 
- Spring boot has layered architecture where one layer communicates with another layer (tier) and their are few layers that exists in the spring boot application.
- Following are the main layers - 
1. Presentation layer
2. Service layer
3. Data access layer


PRESENTATION LAYER -  Presentation layer presents the data and the application features to the user. This is tha layer where in all the controller classes exists.
Controller accept the request from the user, validates the input that the user has given and passes it to the next layer, i.e. service layer.

SERVICE LAYER - Service layer is where the business logic resides in the application. Tasks such as evaluation, decision making, processing of data
is done at this layer. This layer also does the job of interacting with both the presentation layer & data access layer. 

DATA ACCESS LAYER - Data access layer is the layer where all the repository classes resides. These repository classes are responsible for interacting with the databases like performing the 
CRUD operations 

PRESENTATION -----> SERVICE ------> DATA ACCESS 




EXAMPLE  - 
BROWSER -----> CONTROLLER ---> SERVICE ---> REPOSTORY ----> DATABASE (and once the data has been fetched from the cycle) , 
data flow from DATABASE ---> REPOSTORY ---> SERVICE ---> CONTROLLER ----> BROWSER
